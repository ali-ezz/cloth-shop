# Contributing to Cloth Shop

Thank you for your interest in contributing to this project. Contributions help improve the user experience and support future development.

## How to contribute

1. Fork the repository.
2. Create a new branch for your change:
   - `fix/<short-description>` for bug fixes
   - `feature/<short-description>` for new features
   - `docs/<short-description>` for documentation updates
3. Make your changes in the new branch.
4. Commit with a clear message and push the branch to your fork.
5. Open a pull request against `main`.

## Issue guidelines

- Search existing issues before opening a new one.
- Use the appropriate template: bug report or feature request.
- Provide steps to reproduce any bug and include expected behavior.

## Pull request expectations

- Explain the problem and how your change resolves it.
- Keep changes focused and easy to review.
- Update documentation if you add or change functionality.
- Include screenshots or examples when relevant.

## Code style

- Keep HTML, CSS, and JavaScript readable and consistent.
- Use semantic HTML wherever possible.
- Keep CSS modular and scoped to the relevant page.
- Keep JavaScript behavior separated from markup.

## Reviewing and merging

- Pull requests are reviewed before merging.
- Maintain a positive, constructive tone in comments.
- Small, incremental changes are easier to land.
